Um diagrama de classes é usado para descrever a estrutura estática de classes no sistema, permitindo definir os atributos, operações(métodos) e os relacionamentos entre as classes.

Classe {
	é uma representação de um item do mundo real, físico ou abstrato, na forma de um tipo de dados personalizado.
}

Atributos {
	Informações da classe.
}

Método {
	Operações e funções.
}

Classe: Pessoa
Atributos: Altura, Nome, Idade, Peso...
Método: Andar, Comer, Falar...

Representação de uma classe:

Nome: Inclui o nome e o estereótipo da classe (informação sobre a classe)

Atributos: Lista de atributos de classe no formato nome:tipo ou nome:tipo=valor.
tipo é se é int, boolean, string...

Operações: Lista de métodos da classe no formato método(parâmetros):tipo_retorno

Visibilidade dos Membros (Atributos / Métodos)

+ Público 
# Protegido
- Privado
~ Pacote
/ Derivado

exemplo:

      Pessoa 
--------------------
- nome: String
- sobrenome: String
- dataNasc: Date
--------------------
+ calcularIdade(dataNasc): Int
+ estudar(): void


Relacionamento entre Classes

Dependência {
	Dependência fraca, usualmente transiente, que ilustra que uma classe usa informações e serviços de outra classe em algum momento, dependendo dela.

ex:

Classe Carro depende da Classe Roda

representação UML com a seta tracejada (- - - >)
}

Associação {
	Relacionamento mais forte do que a dependência, indica que a classe mantém uma referência a outra classe ao longo do tempo. As assosciações podem conectar mais de duas classes.

ex:

Classe Pessoa tem uma Classe Revista

representação UML com a seta reta (->)
}

Ternária {
	Associação que conecta objetos de três classes. Um losango indica o ponto de convergência(conexão) das classes envolvidas.

representação UML com uma associação em um losango
}

Agregação {
	Indica que uma classe é um contêiner ou uma coleção de outras classes. As classes contidas não dependem do contêiner, assim quando o contêiner é destruído, as classes continuam existindo.

ex:

Classe Departamento possui uma Classe Instrutor

representação UML com a seta reta com a ponta em losango vazado (-<>)

}
